Messages are scraped in reverse chronological order: the most recent message for a given entity (e.g. a group) will be scraped first.

To query the Telegram API, you need to supply your API credentials. These will be sent to the 4CAT server and will be stored there while data is fetched. After the dataset has been created your credentials will be deleted from the server.

The credentials you enter here will also be saved locally to make subsequent dataset creation easier. 4CAT at this time does not support two-factor authentication for Telegram.

You can scrape up to 25 items at a time. Separate the items with commas or line breaks.
